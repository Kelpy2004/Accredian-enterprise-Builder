Next.js officially recommends the App Router for modern project structure, layouts, pages, and routing conventions.

For the bonus API form, use a Next.js Route Handler inside the app directory. Route Handlers are the App Router equivalent of API routes and let you create custom request handlers using the Web Request/Response APIs.

API integration idea

For the mandatory “API integration”, do this:

Option 1: Mock API for programs

Create static data in data/programs.ts and fetch/render it.

Option 2: Better bonus API

Create:

src/app/api/leads/route.ts

When the user submits the lead form, send data to /api/leads.

You can store it temporarily in memory or simply return success:

import { NextResponse } from "next/server";

export async function POST(request: Request) {
  try {
    const body = await request.json();

    if (!body.name || !body.email || !body.company) {
      return NextResponse.json(
        { message: "Name, email and company are required." },
        { status: 400 }
      );
    }

    return NextResponse.json(
      {
        message: "Lead captured successfully",
        lead: body,
      },
      { status: 201 }
    );
  } catch {
    return NextResponse.json(
      { message: "Something went wrong" },
      { status: 500 }
    );
  }
}

This satisfies the bonus: lead capture form + data via API route.
